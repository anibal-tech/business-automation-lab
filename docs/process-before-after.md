# Process Before and After

## Purpose

This document explains how a simple automation can transform a repetitive manual process into a more consistent and visible workflow.

The example used is the task priority report automation.

## Before Automation

The manual process may look like this:

```text
Open spreadsheet
      ↓
Review each task
      ↓
Count priorities manually
      ↓
Identify pending high-priority items
      ↓
Prepare report
      ↓
Send or share summary
```

## Common Problems

Manual reporting can introduce issues such as:

- Time-consuming preparation
- Inconsistent report format
- Human counting errors
- Missed high-priority items
- Delayed visibility
- Repeated manual effort every week

## After Automation

The automated process looks like this:

```text
Update CSV file
      ↓
Run automation script
      ↓
Generate Markdown report
      ↓
Review output
      ↓
Share or publish summary
```

## Improvements

The automated process improves:

- Speed
- Consistency
- Visibility
- Repeatability
- Review quality
- Management follow-up

## What Remains Human

Automation does not remove human judgment.

The user still needs to:

- Review the report
- Validate task context
- Decide next actions
- Communicate with stakeholders
- Adjust priorities when business context changes

## Business Interpretation

This type of automation is useful because it converts operational data into a repeatable management artifact.

The value is not only the script.

The value is the improved process.
